# 如何正确输入 Git 命令 ⚠️

## ❌ 常见错误

当你看到这样的内容时：

```
PS C:\Users\26872\Desktop\心理网站编写\02 管理器> git remote -v
origin  git@github.com:mindcube111/mindcube.git (fetch)
origin  git@github.com:mindcube111/mindcube.git (push)
```

**❌ 错误做法**：复制整行，包括 `PS C:\Users\...>`
```
PS C:\Users\26872\Desktop\心理网站编写\02 管理器> git remote -v
```
这样会导致错误！

## ✅ 正确做法

**只复制命令本身**：
```
git remote -v
```

---

## 📋 正确的命令列表

在 PowerShell 中，**只输入以下命令**（不要包含 `PS C:\...>` 部分）：

### 查看配置
```powershell
git remote -v
```

### 推送代码
```powershell
git push origin main
```

### 查看状态
```powershell
git status
```

### 查看分支
```powershell
git branch
```

### 拉取代码
```powershell
git pull origin main
```

### 添加文件
```powershell
git add .
```

### 提交更改
```powershell
git commit -m "你的提交信息"
```

---

## 🎯 记住这一点

**PowerShell 提示符会自动显示，你只需要输入命令！**

- ✅ 看到 `PS C:\...>` 后，直接输入命令
- ❌ 不要复制包含 `PS C:\...>` 的内容
