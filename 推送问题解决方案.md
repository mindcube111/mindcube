# Git Push 连接问题解决方案

## 问题描述

推送时遇到错误：
```
fatal: unable to access 'https://github.com/...': Recv failure: Connection was reset
```

## 解决方案

### 方案1：重试推送（推荐）

网络连接问题通常是临时的，多试几次：

```powershell
# 重试推送
git push
```

如果还是失败，尝试：

```powershell
# 设置更长的超时时间
git config --global http.postBuffer 524288000
git config --global http.lowSpeedLimit 0
git config --global http.lowSpeedTime 999999

# 再次推送
git push
```

### 方案2：使用 SSH 方式（如果配置了 SSH Key）

如果您的 GitHub 账号配置了 SSH Key，可以改用 SSH：

```powershell
# 查看当前 remote
git remote -v

# 如果 remote 是 HTTPS，改为 SSH
git remote set-url origin git@github.com:yo385042-blip/-psychological-assessment-platform.git

# 推送
git push
```

### 方案3：检查网络连接

1. **检查是否可以访问 GitHub**：
   - 在浏览器中访问：https://github.com
   - 确认可以正常打开

2. **检查代理设置**：
   ```powershell
   # 查看 Git 代理配置
   git config --global --get http.proxy
   git config --global --get https.proxy
   
   # 如果有代理但无法访问，可以取消代理
   git config --global --unset http.proxy
   git config --global --unset https.proxy
   ```

### 方案4：分步推送

如果文件太大，可以尝试：

```powershell
# 只推送当前分支
git push origin main

# 或者指定完整路径
git push https://github.com/yo385042-blip/-psychological-assessment-platform.git main
```

### 方案5：使用 GitHub Desktop 或网页上传

如果命令行一直失败：

1. **使用 GitHub Desktop**：
   - 下载 GitHub Desktop
   - 打开项目
   - 点击 Push 按钮

2. **使用网页上传**：
   - 访问 GitHub 仓库网页
   - 点击 "Upload files"
   - 上传修改的文件

### 方案6：检查防火墙/杀毒软件

某些防火墙或杀毒软件可能阻止 Git 连接：

1. 临时关闭防火墙测试
2. 添加 Git 到防火墙白名单
3. 检查是否有企业网络限制

## 快速重试命令

```powershell
# 1. 重试推送（最简单）
git push

# 2. 如果失败，增加超时时间后重试
git config --global http.postBuffer 524288000
git push

# 3. 如果还是失败，尝试使用 SSH（需要先配置 SSH Key）
git remote set-url origin git@github.com:yo385042-blip/-psychological-assessment-platform.git
git push
```

---

## 验证推送是否成功

推送成功后，可以：

1. **在 GitHub 网页查看**：
   - 访问您的仓库：https://github.com/yo385042-blip/-psychological-assessment-platform
   - 查看最新的提交是否出现

2. **在 Cloudflare Dashboard 查看**：
   - 登录 Cloudflare Dashboard
   - 进入项目 → 部署（Deployments）
   - 应该能看到新的部署自动开始

---

**建议**：先尝试方案1（重试几次），如果多次失败再尝试其他方案。
