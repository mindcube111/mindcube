# 🚀 快速操作指南

根据你当前的 Cloudflare Dashboard 截图，你已经创建了 KV 命名空间 `mindcube`。

## 📍 你现在的位置

- ✅ 在 Cloudflare Dashboard
- ✅ KV 命名空间已创建：`mindcube`
- 📋 下一步：绑定到 Pages 项目

---

## 🎯 接下来的 3 个步骤

### 步骤 1️⃣：找到或创建 Pages 项目

**在左侧菜单栏：**

1. 找到 **"构建" (Build)** 部分
2. 展开它，找到 **"Pages"** 并点击
   - 如果看到项目列表，选择一个项目
   - 如果没有项目，点击 **"Create a project"（创建项目）**

---

### 步骤 2️⃣：进入设置页面

1. 点击你的 **Pages 项目名称**（或创建新项目）
2. 在项目详情页，点击顶部的 **"Settings"（设置）** 标签
3. 在左侧菜单中找到并点击 **"Functions"**

---

### 步骤 3️⃣：绑定 KV 命名空间

在 Functions 设置页面：

1. **向下滚动**，找到 **"KV namespace bindings"** 部分

2. 点击 **"+ Add binding"（添加绑定）** 按钮

3. 填写信息：
   ```
   Variable name: DB
   KV namespace: mindcube（从下拉菜单选择）
   ```

4. 点击 **"Save"（保存）**

5. （可选）在 **"Preview"** 标签下，重复上述步骤添加预览环境绑定

---

## ✅ 完成后你应该看到

```
KV namespace bindings
├── Production
│   └── Variable name: DB
│       └── KV namespace: mindcube
```

---

## 🆘 如果找不到 Pages

**如果左侧菜单中没有 "Pages"：**

1. 可能需要先创建一个项目
2. 点击 **"+添加"** 按钮
3. 选择 **"Pages"**
4. 然后按照创建项目的流程操作

---

## 📝 创建 Pages 项目（如果还没有）

1. 点击 **"Create a project"** 或 **"+添加"** > **"Pages"**

2. 选择连接方式：
   - **Connect to Git**（连接 Git） - 推荐
   - **Upload assets**（上传文件） - 手动

3. 如果选择 Git：
   - 授权 GitHub/GitLab
   - 选择你的仓库
   - 配置：
     - Project name: `psychological-assessment-platform`
     - Build command: `npm run build`
     - Build output directory: `dist`
   - 点击 **"Save and Deploy"**

4. 项目创建后，再回到步骤 2 和 3 绑定 KV

---

## 🎉 完成后继续

KV 绑定完成后，继续：

1. 配置环境变量 `JWT_SECRET`
2. 部署项目
3. 初始化管理员账号

---

## 💡 重要提示

- **变量名必须完全一致**: `DB`（大写字母）
- 绑定后需要重新部署项目才能生效
- 可以同时绑定生产环境和预览环境
